FeedHenry RainCatcher?

At the top-level, RainCatcher is a set of NodeJS modules that are used to develop workforce management (WFM) applications.

RainCatcher modules are packaged and distributed via npm. They are designed to be included in your application using Browserify. RainCatcher modules export one or more of:

  • Angular.js directives or services providing client-side functionality for both the mobile and portal clients.
  • Express.js routes providing a REST API to be consumed by the client-side portions of the module.
  • FeedHenry sync configurations for enabling data synchronization of a module's data.

Each module's README file includes more detail on their purpose and how to use them.

Some of the modules for RainCatcher include:

  • Visual components
    • wfm-signature for capturing signatures
    • wfm-map for integrating with google maps
  • Angular Services and Directives
    • wfm-workflow for moving work orders through workflows
    • wfm-workorder for work orders
    • wfm-user for managing users, groups, and authorisation
    • wfm-risk-assessment for assessing the risk at a job site
    • wfm-vehicle-inspection for vehicle inspection
    • fh-rc-schedule for job scheduling
  • Low-level modules
    • wfm-sync for data synchronization
    • wfm-appform for integrating with RHMAP appforms
    • wfm-mediator an implementation of the mediator pattern
